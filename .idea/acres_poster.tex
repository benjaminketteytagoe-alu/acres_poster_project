\begin{itemize}
  \item \textbf{Aim:} Build a centralized web platform to digitize property manageement and lease compliance.
  \item \textbf{Objective 1:} Design a multi-unit dashboard for financial and occupancy oversight.
  \item \textbf{Objective 2:} Implement a secure digital vault for lease agreements and automated invoicing.

\block{4. Constraints and Design Decisions}{
Posters are stronger when they show real engineering trade-offs.

\begin{itemize}
  \item \textbf{Constraints:} Must be mobile-responsive for low-bandwidth environments; must comply with the 2021 Rwanda Data Protection Act
  \item Data quality, privacy, security, or compliance issues
  \item \textbf{Decisions:} Chose Next.js and MySQL for high performance and structured data integrity.
  \item \textbf{Trade-off:} Optimized for simplicity and low barrier to entry (QR Codes) over complex features that would require tenant app downloads
\end{itemize}
}

Acres is an "Administrative Engine" that transforms property management from notebooks to a digital dashboard. Landlords upload leases and track rent, while tenants use unit-specific QR codes to trigger maintenance workflows. The system centralizes data, providing automated alerts and transparent financial history.

\block{8. Impact, Contribution, and Learning}{
\textbf{Frame the contribution at three levels.}

\begin{itemize}
  \item \textbf{Technical:} Integrated a QR-to-Web workflow that eliminates the need for tenant-side authentication while maintaining data accuracy.
  \item \textbf{User:} Landlords gain financial visibility; tenants receive documented proof of payments and faster maintenance response.
  \item \textbf{Research:} Validated that small-scale landlords prioritize "legal security" over "complex accounting features."
  \item \textbf{Mission:} Supports Rwanda’s National Housing Policy by professionalizing the informal rental sector through accessible tech.
\end{itemize}

\takeawaybox{Do not just show that it works. Show why it matters.}
}

\block{9. Limitations and Next Steps}{
\textbf{Strong projects acknowledge what is still unfinished.}

\begin{itemize}
  \item \textbf{Limitaions:} Currently relies on manual input for payment verification rather than direct mobile money (Momo) API integration.
  \item \textbf{Assumptions:} Assumes landlords have basic digital literacy and consistent smartphone access.
  \item \textbf{Next Steps:} Integrate local payment gateways (MTN/Airtel) for real-time automated rent reconciliation.
\end{itemize}
}
